IT基盤刷新プロジェクト

オンプレミス環境からAWSクラウドへの包括的移行計画

プロジェクト概要

サン・クロレラジャパン株式会社様の既存オンプレミス環境を、Amazon Web Services（AWS）クラウドプラットフォームへ移行する包括的なインフラストラクチャをご提案いたします。セキュリティ、可用性、拡張性を重視した設計と、Windows 10サポート終了対応、PC環境刷新を含む包括的なソリューションにより、機能要件を満たしながらコスト効率を最大化し、御社のIT基盤を最適な状態へと発展させます。

※AWS移行をご提案する詳細な経緯については、別紙「クラウド移行によるIT基盤刷新のご提案」をご参照ください。

    Windows 10サポート終了対応

    Microsoft Windows 10のサポート終了（2025年10月14日）に伴い、移行完了までの期間における暫定的なセキュリティ強化対策を本プロジェクトに含めて実施いたします。

    暫定セキュリティ強化対応内容

    • EDR/MDRセキュリティ監視強化
    • ネットワークセグメンテーション強化
    • 仮想パッチ適用（IPS/IDS）
    • 24時間セキュリティ監視体制
    • インシデント対応体制構築
    適用期間
    2025年10月15日～
    移行完了まで（最大2.5ヶ月）

    重要事項

    本対応は、9月1日からの作業開始を前提として、追加予算なしで実施いたします。プロジェクト予算内での包括的なセキュリティ対応として提供します。

    リスク管理・対策

    VPN接続・ネットワーク障害リスク

    対策：Site-to-Site VPN + Client VPNの冗長構成、Transit Gatewayによる複数経路確保、ネットワーク監視・自動復旧機能

    EC2・FSx可用性リスク

    対策：EC2予備インスタンス（停止状態）準備、FSx for Windowsマルチ可用性ゾーン構成、CloudWatchによる自動監視・アラート

    認証・アクセス制御リスク

    対策：AWS IAM Identity Centerによる統合認証、多要素認証強制、アクセスログ監視・異常検知

    データ保護・バックアップリスク

    対策：S3 + Glacier階層化バックアップ、Kinesis Data Firehoseによるリアルタイムログ保存、定期復旧テスト実施

    外部SaaS接続・依存リスク

    対策：Office365・Salesforce・Amazon Connect等への接続監視、認証連携障害時の代替手順、SaaS側障害時の業務継続計画

    移行・切替リスク

    対策：段階的移行による影響最小化、複数回リハーサル実施、Route53による段階的DNS切替、ロールバック手順完備
